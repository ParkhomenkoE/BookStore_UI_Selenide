# BookStore_UI_Selenide
# QA Automation Project – BookStore

## Project Overview

This project demonstrates end-to-end testing of a BookStore web application:
https://demoqa.com/books

It covers UI automation, API testing, BDD approach, and test management, simulating a real QA workflow.

## Tech Stack

* Java
* Selenide (UI automation)
* RestAssured (API testing)
* Cucumber (BDD)
* TestNG / JUnit
* Gradle
* Qase (test management)

## Test Coverage

### UI Testing

* Login (valid / invalid scenarios)
* Book search
* Add to cart
* Remove from cart
* Checkout process

### API Testing

* GET books
* POST user/login
* DELETE book
* Negative scenarios

### BDD Testing

* User scenarios described in Gherkin
* Feature files + step definitions

## Test Types

* Functional testing
* UI testing
* API testing
* Regression testing
* Basic security testing

## Project Structure

* `pages` – Page Object classes
* `tests` – Test classes
* `utils` – Helper methods
* `api` – API tests
* `features` – Cucumber feature files

## How to Run Tests

### UI tests
./gradlew test
### API tests
./gradlew apiTest
